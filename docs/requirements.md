# アンケート画像解析システム 要件定義書

## 概要

本システムは、スキャンされたアンケート用紙画像から手書き文字や選択内容を読み取り、自動的にデジタル化（CSV形式）するローカル環境向けWebアプリケーションである。

ユーザーはサンプル画像に対してOCR対象範囲（回答欄）をGUIで設定でき、設定内容に基づいて複数の回答画像から情報を抽出し、CSV形式で保存・集計できる。

---

## ハードウェア・実行環境

- **CPU**: Intel Core-i7 12th（デスクトップ版）
- **メモリ**: 32GB（DDR5）
- **OS**: Windows 11 Home
- **仮想環境**: WSL2 + Docker Desktop for Windows

---

## 技術スタック

- **フロントエンド**: React.js（TypeScript）, MUI, react-konva
- **バックエンド**: Python（FastAPI 等想定）
- **OCRエンジン**: Tesseract OCR（日本語対応）

## 機能要件

### アップロード機能

- サンプル画像のアップロード
- 複数の回答済みアンケート画像のアップロード（`.jpg`, `.png`）

### OCR範囲設定機能（React GUI）

- 画像上で矩形選択によるOCR範囲の指定（react-konva使用）
- 選択ごとに以下を設定可能：
  - `ID`：各設問の識別子
  - `回答方式`：以下のいずれかを手動で指定
    - **記述式**（テキスト）
    - **選択式（単一／複数選択）**
    - **表形式（リッカート式）**
- 範囲情報は以下の形式で保存：
  - **JSON**（`/storage/map` に保存／読み込み可）
- 表形式においては複数の選択肢範囲をグループ化して登録可能

### 解析機能

- 複数画像を対象に、以下の内容を自動抽出：
  - 記述式：TesseractによるOCR文字認識
  - 選択式・リッカート式：画像処理（OpenCV）により囲みや塗り潰しを判定（輪郭検出）
- 抽出された内容は、JSON定義に従って自動的に設問IDへマッピング

### 結果出力機能

- 出力形式：CSV（`/storage/output` に保存）
- 出力構成：
  - ヘッダー：`回答ID, 設問1, 設問2, ...`
  - 回答IDは画像ファイルごとに連番で付与（例：001, 002, ...）

### 並列バッチ処理（追加要件）

- 回答済みアンケート画像は複数ファイルをアップロードでき、バックエンドで **並列処理** を行う
- 並列処理は1画像＝1タスクとして扱い、OCR・マーク認識を独立に実行
- 並列処理方式：`concurrent.futures` による `ThreadPoolExecutor` または `ProcessPoolExecutor`
- 使用するスレッド数・プロセス数は `os.cpu_count()` に基づいて自動で決定
- 処理結果はCSV形式で `/storage/output/` に出力（ファイル名にタイムスタンプ付き）

## 非機能要件

- **セキュリティ**：
  - 現時点ではWeb公開予定なし。ただし、将来の公開に備えて最低限の対策を実施（CSRF/XSS, ファイルバリデーション等）
- **対応画像形式**：
  - `.jpeg`, `.png` のみ対応（**PDFは将来対応**）
  - **1ファイル = 1アンケート**（複数枚対応は将来的に拡張予定）
- **実行速度**：
  - ローカル環境での実行前提のため、高負荷処理も許容範囲内
- **費用**：
  - すべて無料OSSで構成する

---

## 画面構成（ステップ構成）

1. サンプル画像のアップロード
2. 回答欄設定（矩形選択、ID/形式指定）
   - または事前に保存された JSON 設定ファイルの読み込み
3. 回答画像のアップロード（複数枚）
4. 解析開始
5. 出力ファイル（CSV）の生成・保存

| 分類     | OSS名                   | 備考                                  |
| -------- | ----------------------- | ------------------------------------- |
| OCR      | Tesseract OCR           | 日本語対応、CLI/APIあり               |
| 画像処理 | OpenCV                  | 範囲内の囲み検出等                    |
| Web UI   | React, MUI, react-konva | 範囲選択インタフェース構築に使用      |
| サーバ   | FastAPI                 | Python製軽量APIフレームワーク（予定） |

## 補足事項（仕様的補強）

- 回答方式の推定は手動指定が前提。
- OCR範囲定義はJSONで保存し、再利用可能。
- チェック／選択式の判定は**輪郭（囲み）の検出**を基本とする。
- ファイル名順に回答IDを連番で付与。
